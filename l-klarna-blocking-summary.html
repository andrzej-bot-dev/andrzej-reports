🔴 The Problem in 30 Seconds

l.klarna.com is Klarna's branded short link domain used in marketing emails, push notifications, and the "Sign in" button on klarna.com. It doesn't resolve directly — it has a CNAME chain:

CNAME l.klarna.com What users see
CNAME getklarna.customlinks.appsflyer.com 🚫 This is what gets blocked
CNAME onelinkalbnew.onelink.me.cdn.cloudflare.net AppsFlyer OneLink CDN
A 104.18.39.175 / 172.64.148.81 Cloudflare
Root cause: AppsFlyer (a mobile attribution/tracking platform, classified as a data broker) is one of the most aggressively blocked domains on the internet. Any DNS blocker with CNAME uncloaking follows the chain from l.klarna.com to appsflyer.com and blocks it.

🎯 Revised Block Rate Estimate

After adversarial review (cross-checked against Claude's independent analysis), the estimate was revised downward from the initial 2.6–10% to a more defensible range:

ScenarioEst. Users% of 119M
Conservative (NextDNS + AdGuard DNS confirmed only)~1.2M~1.0%
Central (+ AdGuard Home + ControlD + some RethinkDNS)~1.5–3.5M~1.3–3.0%
Upper bound (+ corporate DNS + all edge cases)~5M~4.2%

Why the estimate dropped from initial 2.6%/10%

  1. Pi-hole doesn't block it. 2–5M Pi-hole installations were originally counted as affected. They're not — Pi-hole doesn't follow CNAME chains, and l.klarna.com isn't on any list directly. Removes ~2–5M users from the estimate.
  2. Browser extensions don't block it. uBlock Origin, EasyPrivacy, and Firefox Disconnect all have zero appsflyer entries. The "10% including browser" figure was wrong. Removes ~5–8M users.
  3. Brave Aggressive is not default. CNAME uncloaking in Brave requires opt-in to Aggressive mode. Most Brave users are on Standard. Removes most Brave users.
  4. HaGeZi Normal (most common level) has zero appsflyer entries. Only Pro/Pro++/Ultimate levels include them. Many HaGeZi users are on Normal. Reduces HaGeZi impact.
What still holds: NextDNS (confirmed by Klarna engineer) and AdGuard DNS (confirmed by Klarna engineer in issue #2089) both actively block l.klarna.com today. These are not estimates — they're verified facts. Combined, these services represent millions of users in Klarna's core markets.

⚔️ Adversarial Review — Claude's Critique vs Reality

An independent analysis by Claude (Anthropic) challenged several claims. Here's an honest scorecard:

Claim: "19,603 looks fabricated" WRONG

Verified live: curl -sL ".../ultimate.txt" | grep -ci appsflyer → 19,603. Breakdown: 64 appsflyer.com subdomains + 19,539 appsflyersdk.com subdomains. The number is real, reproducible, and not fabricated. Anyone can verify in 5 seconds.

Claim: "HaGeZi allowlist protects most users" WRONG

l.klarna.com is NOT in any HaGeZi compiled output (multi.txt, pro.txt, normal.txt, ultimate.txt) — confirmed by grep returning exit code 1. The allowlist concern was about the referral sub-list, which was a side issue. The actual blocking path is StevenBlack → NextDNS and AdGuard DNS filter CNAME-cloaking detection, not HaGeZi.

Claim: "Brave CNAME uncloaking is not default" CORRECT

Brave's wiki confirms: CNAME uncloaking is only in Aggressive mode, which is opt-in. Standard mode (default) does not uncloak. The original report's "10% including browser-level" was overstated. This has been corrected.

Claim: "Pi-hole users mostly unaffected" CORRECT

Pi-hole does not follow CNAME chains by default. Since l.klarna.com is not on any blocklist directly, Pi-hole resolves it fine. Only Pi-hole setups forwarding to NextDNS/AdGuard DNS upstream would be affected. This has been corrected.

Claim: "Claude's 0.2% estimate" UNDERSTATED

Claude estimated 0.1–0.2% (~120–240K users). This is likely too low. AdGuard DNS alone has 15–30M users (confirmed by AdGuard's own website showing 91M+ users across all products). Even if only 10% of AdGuard DNS users are in Klarna markets and have appsflyer blocked via CNAME detection, that's 150K–300K from AdGuard alone — already exceeding Claude's upper bound. Adding NextDNS (confirmed blocking) pushes it higher.

Bottom line: Both analyses have merit. The truth is between them — likely 1–3% of Klarna users affected. The original report leaned too heavily on Pi-hole/browser numbers that don't hold up; Claude's estimate undercounts the confirmed NextDNS + AdGuard DNS blocking.

💡 Bottom Line

l.klarna.com is blocked for an estimated 1–3% of Klarna's 119M users (~1.2–3.5M people), primarily through NextDNS and AdGuard DNS which both have confirmed CNAME-based blocking of the AppsFlyer chain. The original estimate of 2.6–10% was too high because it counted Pi-hole users (not affected — no CNAME following) and browser extensions (not affected — no appsflyer in EasyPrivacy/Disconnect).


The problem is real but narrow: it affects users of DNS blockers with CNAME uncloaking technology (NextDNS, AdGuard DNS, ControlD, AdGuard Home). The fix is straightforward — Klarna needs either a server-side proxy (A record instead of CNAME) or per-alias exceptions like AdGuard already grants for PayMaya.
