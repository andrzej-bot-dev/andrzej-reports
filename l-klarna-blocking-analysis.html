🔗 The CNAME Chain — Root Cause

The entire blocking problem stems from a single DNS configuration. When a user clicks a Klarna marketing link (email, SMS, push notification, social media), the URL points to l.klarna.com. But this domain doesn't resolve directly — it chains through AppsFlyer:

CNAME l.klarna.com User-facing URL (Klarna's branded domain)
↓
CNAME getklarna.customlinks.appsflyer.com 🚫 BLOCKED by 7/8 major blocklists
↓
CNAME onelinkalbnew.onelink.me.cdn.cloudflare.net AppsFlyer OneLink CDN via Cloudflare
↓
A 104.18.39.175 / 172.64.148.81 Cloudflare anycast IPs
What each layer does:
  • l.klarna.com — What users see. Klarna's first-party branded short link domain.
  • getklarna.customlinks.appsflyer.com — AppsFlyer processes the click, logs analytics (attribution, device fingerprinting), and determines redirect destination. The getklarna prefix is a vanity label for Klarna on AppsFlyer's infrastructure.
  • onelinkalbnew.onelink.me — AppsFlyer's OneLink deep-linking product, proxied through Cloudflare's CDN.

🔍 CNAME Uncloaking Technology

CNAME uncloaking (also called "CNAME following" or "CNAME de-cloaking") is a technique where a DNS resolver follows the CNAME chain of a domain and checks every domain in the chain against blocklists. This defeats the "first-party cloaking" trick used by tracking vendors like AppsFlyer, Criteo, and Adobe.

About AppsFlyer

AppsFlyer is an Israeli mobile attribution and marketing analytics company, founded in 2011. It is classified by Wikipedia as a data broker. Key facts:

  • 100,000+ apps use the AppsFlyer SDK
  • Market position: #1 or #2 mobile attribution globally (competing with Adjust, Branch)
  • Subject of a 2025 NOYB GDPR complaint (with TikTok) for illegal data sharing
  • March 2026 SDK malware incident — malware found in SDK replacing Bitcoin wallet addresses
  • These controversies ensure aggressive listing on privacy-focused blocklists

🗓️ Timeline: The GitHub Issue That Exposed Everything

July 22, 2026 — A Klarna engineer going by v-ilinskiy-klarna opened issue #10901 on the HaGeZi DNS Blocklists GitHub repo:

"l.klarna.com is Klarna's first-party branded domain for AppsFlyer OneLink — it is not a third-party ad-tracking redirector, it's Klarna's own infrastructure used to deep-link Klarna's own customers into the Klarna app or web experience."

The engineer confirmed that NextDNS's "Ads & Blocklist" blocks l.klarna.com today, and that links behind l.klarna.com fail to open the Klarna app — including the critical "Sign in" button on klarna.com.

HaGeZi's response was revealing:

"The domain l.klarna.com is not directly included in the NextDNS list, but it is blocked via the CNAME getklarna.customlinks.appsflyer.com (appsflyer.com is completely blocked) which is correctly classified as a tracking domain. I don't know of any other lists that use my referral blacklist as a source — I'd say none. The problem is that l.klarna.com uses an AppsFlyer tracker, which is widely blocked in lists based on the root domain appsflyer.com."

This confirms our analysis: the blocking is driven by appsflyer.com, not by any list targeting Klarna.

  • Move off AppsFlyer: Given the NOYB GDPR complaint and March 2026 SDK malware incident, the reputational risk of AppsFlyer association is increasing
